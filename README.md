### 브랜치 전략

- Git-flow 전략을 기반으로 main, develop 브랜치와 feature, 버그 fix를 위한 fix 보조 브랜치를 운용했습니다.
- main, dev, feature 브랜치로 나누어 개발을 하였습니다.
    - **main** 브랜치는 배포 단계에서만 사용하는 브랜치입니다.
    - **dev** 브랜치는 개발 단계에서 git-flow의 master 역할을 하는 브랜치입니다.
    - **feature** 브랜치는 기능 단위로 독립적인 개발 환경을 위하여 사용하고 merge 후 각 브랜치를 삭제해주었습니다.
